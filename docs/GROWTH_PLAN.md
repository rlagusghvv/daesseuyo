# 대쓰요 성장 방향

## 목표

실제 트래픽을 늘리는 1차 목표는 신규 사용자가 10초 안에 한 판을 시작하고, 게임이 끝난 뒤 결과를 공유하게 만드는 것이다. 광고 집행보다 먼저 제품 내부의 유입 루프를 만든다.

## 우선순위

1. 공개 링크 품질
   - 검색/메신저 미리보기용 title, description, OG, Twitter 메타를 유지한다.
   - `/robots.txt`, `/sitemap.xml`, `/manifest.webmanifest`를 유지한다.
   - 공유 URL에는 `ref` 값을 붙여 유입 경로를 구분한다.

2. 첫 플레이 전환
   - 첫 화면은 `AI 대전`, `오늘의 승부`, `랭크 테스트`, `친구랑 하기` 네 개만 둔다.
   - 규칙 설명은 3문장 이내로 유지한다.
   - 모바일에서 스크롤 없이 바로 Q/W/E/R 버튼이 보여야 한다.

3. 공유 루프
   - 게임 종료 후 결과 공유 버튼을 노출한다.
   - 공유 문구는 점수, 투구 수, 읽기율을 포함한다.
   - 친구 초대 링크는 `?duel=방코드&ref=invite` 형태로 만든다.

4. 리텐션 루프
   - `오늘의 승부`는 매일 같은 이름의 진입점으로 유지한다.
   - 랭크 기록은 기기 기반으로 유지하되, 로그인 전까지는 가볍게 둔다.
   - 다음 단계는 일간 최고 기록과 7일 연속 플레이 표시다.

5. 최소 지표
   - `/metrics`는 view, start, invite, share, game_over만 받는다.
   - `/traffic`으로 최근 24시간과 7일 요약을 확인한다.
   - 개인 식별값은 SHA-256 해시 앞부분만 저장한다.

## 홍보 실행안

### 1차

- 야구 커뮤니티에는 긴 소개보다 한 문장과 링크만 쓴다.
- 예시: `9회말 2아웃에서 투수/타자 심리전만 남긴 웹 야구게임 만들어봤습니다. 1분 컷입니다.`
- 결과 공유 이미지는 아직 만들지 않는다. 먼저 텍스트 공유 전환율을 본다.

### 2차

- 쇼츠/릴스는 플레이 화면 녹화 8초 버전으로 만든다.
- 구성: `9회말 2아웃` -> `Q/W/E/R 선택` -> `홈런/삼진 결과` -> URL.
- 영상보다 링크 클릭과 완주율이 먼저다.

### 3차

- 일간 챌린지 기록판을 추가한다.
- 특정 날짜의 최고 읽기율, 최단 투구 수, 끝내기 성공 여부를 공유하게 만든다.
- 그때부터 커뮤니티 게시글 제목은 `오늘의 대쓰요 승부`로 고정한다.

## 다음 제품 과제

1. `/leaderboard` 또는 `/daily` 기록 API.
2. 결과 공유 카드 이미지 생성.
3. 첫 3구 튜토리얼 모드.
4. 초대받은 사용자의 역할 선택 개선.
5. 플레이 종료 후 바로 재도전하는 흐름 개선.
